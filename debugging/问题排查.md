根因

  Redis key char:live_room:room_content:v2:catalog_version 值被污染为 "386\n"（带换行符），CAS Lua
  脚本做字符串比较永远失败。来源是某次手动 echo 386 | redis-cli -x SET ...，echo 默认追加 \n。

  排查过程中犯的错误

  1. 无脑怀疑代码逻辑 — 假设是 retry 逻辑 bug、version 计算错误、前端传参不对，花大量时间读改代码
  2. Codex 犯同样错误 — 认为是前端问题，在前端做了修改（已回滚）
  3. 忽略错误信息中的关键线索 — actual_in_redis="386\n" 第一次出现就是最终答案，但被忽略了
  4. 没区分"代码写入"和"人工写入" —
  所有代码路径（strconv.FormatInt、INCR）不可能产生换行符，唯一来源是外部手动操作
